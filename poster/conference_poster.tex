\begin{multicols}{2}

\section{Introductie}
Bedrijven zijn wettelijk verplicht om de identiteit van hun klanten te verifiëren (AML-wetgeving). 
De huidige methode bij Scrada vereist vaak manuele invoer van KBO-gegevens, wat de efficiëntie vertraagt.

\begin{tcolorbox}[colback=white, colframe=hogent-blue, arc=8pt, title=Centrale Onderzoeksvraag]
\textit{“Hoe kan het KYC-proces bij de identiteitsverificatie van bedrijven op het Peppol-platform van Scrada geautomatiseerd worden op een GDPR-conforme, fraudeveilige en kostenefficiënte manier?”} 
\end{tcolorbox}

\section{De Oplossing}
Vanwege de hoge abonnementskosten van de officiële KBO-API (€50 per 2000 requests), is gekozen voor een kostenloze PoC:
\begin{itemize}
    \item \textbf{Bedrijfsstatus:} Live validatie via de gratis VIES-API van de Europese Commissie[cite: 9].
    \item \textbf{Bestuurdersgegevens:} Een custom .NET scraper die de publieke KBO-database raadpleegt door middel van HtmlAgilityPack[cite: 10].
    \item \textbf{Identiteitsmatch:} Vergelijking van gescrapte namen met OCR-data van de eID, samen met biometrische verificatie[cite: 11].
\end{itemize}

\section{Systeemarchitectuur}
De applicatie is gebouwd met een Flutter frontend en een ASP.NET Core backend.

\section{Biometrische Verificatie}
Om identiteitsfraude tegen te gaan, is het gebruik van biometrie essentieel om de indentiteit van de gebruiker te verifiëren:
\begin{itemize}
    \item \textbf{Gezichtsherkenning:} Een biometrische vergelijking tussen een live-selfie en de foto op de gescande eID zorgt voor de authenticatie van de persoon[cite: 13].
    \item \textbf{Liveness Detection:} Voorkomt fraude door te bevestigen dat een fysiek persoon aanwezig is[cite: 14].
    \item \textbf{Bevoegdheidscheck:} De geverifieerde persoon wordt in de backend gematcht met de lijst van statutaire bestuurders die live uit de KBO is gescrapt[cite: 15].
\end{itemize}

\section{Resultaten \& Verificatie}
De PoC bewijst dat een volledige identiteitscheck van een bedrijf en bestuurder slechts enkele seconden kan duren:
\begin{itemize}
    \item \textbf{Naadloze Match:} Succesvolle koppeling tussen de live KBO-data en de biometrische face-match uit de app[cite: 16].
    \item \textbf{Data-cleaning:} De backend vertaalt ruwe HTML-namen naar tekst, wat essentieel is voor een betrouwbare vergelijking met de eID[cite: 17].
    \item \textbf{Full-Stack Flow:} Een sluitende keten van bedrijfsstatus (VIES \& KBO), documentcontrole (OCR) en biometrie[cite: 18].
\end{itemize}

\section{Conclusie}
Deze PoC bewijst dat manuele KYC-processen bij Scrada verleden tijd zijn[cite: 19]. 
Door een Flutter-app te koppelen aan een .NET-backend, ontstaat een flow die zowel de snelheid voor de gebruiker verhoogt als de veiligheid van de data garandeert[cite: 20]. 
Hoewel de web-scraper een ideale methode was om deze PoC kosteloos te ontwikkelen, is een overstap naar de officiële KBO-API de volgende stap voor een optimale productieomgeving[cite: 21]. 
De huidige architectuur is hier al op voorbereid: de scraper code kan simpelweg worden ingewisseld voor de officiële API-koppeling[cite: 21].
